# 📊 任务层级系统

Priaxis 采用四层任务层级系统，模拟 RPG 游戏的主线-章节-副本-任务结构。

---

## 层级结构

```
主线剧情（Season）     →  人生阶段/身份状态 (1-5年)
    ↓
篇章（Chapter）        →  阶段目标（必须完成才能通关） (3-12个月)
    ↓
副本剧情（Quest）      →  项目企划（可选） (1周-3个月)
    ↓
每日任务（Task）       →  具体可执行的行动
```

---

## 🎭 主线剧情 (Season)

代表人生的一个重要阶段或身份状态。

**示例**:
- "博士研究生阶段" (2024-2028)
- "创业者之路" (2025-2027)
- "健康重塑计划" (2025)

**属性**:
- 标题、描述、起止日期
- 包含多个篇章
- 整体进度由篇章进度聚合

---

## 📖 篇章 (Chapter)

主线下的阶段性目标，**必须完成**才能推进主线。

**示例**:
- "完成 NeoQA 论文投稿" (3个月)
- "通过博士资格考试"
- "建立个人技术博客" (6个月)

**特点**:
- 有明确的完成标准
- 进度由关联副本/任务聚合
- 完成后可解锁称号和奖励

---

## ⚔️ 副本 (Quest)

可选的项目或任务集，关联到篇章。

**示例**:
- "修改 MuSiQue prompt 实验"
- "学习 React Native"
- "准备技术分享"

**属性**:
- 重要性: 高/中/低 (影响任务排序)
- 进度: 由关联任务自动计算
- 可设置解锁日期和截止日期

---

## ✅ 任务 (Task)

最小的可执行单元，具体的行动项。

**四象限分类** (艾森豪威尔矩阵):

| | 紧急 | 不紧急 |
|---|---|---|
| **重要** | 🔴 立即做 | 🟡 计划做 |
| **不重要** | 🟠 委托/快速处理 | ⚪ 考虑删除 |

**属性**:
- 重要性/紧急性/努力量
- 截止日期 (红绿灯显示)
- 关联副本 (自动继承重要性)
- 番茄钟计数

---

## 🔄 进度传播

```
任务完成 → 自动更新副本进度 → 自动更新篇章进度 → 更新主线进度
```

- 副本进度 = 已完成任务数 / 总任务数
- 篇章进度 = 已完成副本数 / 总副本数
- 主线进度 = 已完成篇章数 / 总篇章数
